## Overview

THAMES-Hydration is a C++ program that simulates cement hydration by coupling:
- **Kinetic dissolution models** (Parrot-Killoh, Standard, Pozzolanic)
- **GEMS3K thermodynamic calculations** for phase equilibrium
- **3D microstructure evolution** using a lattice model

The program supports three simulation types:
1. **HYDRATION** (simtype=2) - Standard cement hydration
2. **LEACHING** (simtype=3) - Leaching simulation
3. **SULFATE_ATTACK** (simtype=4) - External sulfate attack

### Source Code Location

```
backend/thames-hydration/src/
├── thames.cc              # Main program entry point
├── thames.h               # Main program header
└── thameslib/             # Core library
    ├── ChemicalSystem.cc/h    # GEMS interface, phase definitions
    ├── Controller.cc/h        # Simulation orchestration
    ├── KineticController.cc/h # Kinetic model management
    ├── KineticModel.cc/h      # Base kinetic model class
    ├── KineticData.h          # Kinetic parameter structure
    ├── ParrotKillohModel.cc/h # Clinker phase kinetics
    ├── StandardKineticModel.cc/h  # Sulfate phase kinetics
    ├── PozzolanicModel.cc/h   # Pozzolan kinetics
    ├── Lattice.cc/h           # 3D microstructure
    ├── Site.cc/h              # Individual voxel
    ├── Interface.cc/h         # Growth/dissolution interfaces
    ├── global.h               # Constants and type definitions
    └── Exceptions.cc/h        # Error handling
```

## Program Flow

### Main Program (thames.cc)

```
main()
├── checkArgs()              # Parse command line: -v, -s, -x, -o folder
├── User inputs simulation type (1=quit, 2=hydration, 3=leaching, 4=sulfate)
├── User inputs GEM database file path (thames-dat.lst)
├── ChemicalSystem()         # Parse GEMS + simparams.json
├── RanGen()                 # Initialize random number generator
├── User inputs microstructure file path
├── Lattice()                # Load initial microstructure
├── [If sulfate attack: ThermalStrain(), AppliedStrain()]
├── KineticController()      # Parse kinetic data from simparams.json
├── User inputs output file root name
├── prepOutputFolder()       # Create output directory, copy input files
├── Controller()             # Initialize simulation controller
├── writeReport()            # Write simulation parameters report
└── Ctrl->doCycle()          # RUN THE SIMULATION
```

### Input Sequence (stdin)

The program reads the following inputs in order via stdin:

```
1. Simulation type (integer: 2=hydration, 3=leaching, 4=sulfate)
2. GEM input file name (e.g., "thames-dat.lst")
3. Simulation parameter file name (e.g., "simparams.json")
4. Microstructure file name (e.g., "ThamesFA-200.thames.img")
5. Output file root name (e.g., "ThamesFA")
```

This can be automated via:
```bash
echo -e "2\nthames-dat.lst\nsimparams.json\nmicrostructure.img\noutput" | ./thames
```

### Command Line Options

```
thames [--verbose|-v] [--suppress|-s] [--xyz|-x] [--outfolder|-o] folder [--help|-h]

  --verbose (-v)      Produce verbose output
  --suppress (-s)     Suppress warning messages
  --xyz (-x)          Create 3D visualization movie
  --outfolder (-o)    Name of folder for output data (default: "Result")
```

## Core Classes

### ChemicalSystem

**Purpose:** Interface between GEMS thermodynamic engine and THAMES microstructure model.

**Key Responsibilities:**
- Parse GEMS database files
- Parse simparams.json for phase definitions
- Map THAMES phases to GEMS phases and DCs
- Run GEMS equilibrium calculations
- Track phase masses, volumes, moles

**Constructor:**
```cpp
ChemicalSystem(const std::string &GEMfilename,      // e.g., "thames-dat.lst"
               const std::string &jsonFileName,     // e.g., "simparams.json"
               const bool verbose,
               const bool warning = false,
               bool testSimparams = false);
```

**Key Methods:**
- `parseDoc()` - Master JSON parsing
- `parseMicroPhases()` - Parse phase definitions
- `parseGEMPhaseData()` - Parse GEM phase linkages
- `parseKineticData()` - Parse kinetic parameters
- `calculateState()` - Run GEMS equilibrium calculation

### KineticController

**Purpose:** Manages all kinetic models and calculates dissolution rates.

**Constructor:**
```cpp
KineticController(ChemicalSystem *cs,
                  Lattice *lattice,
                  const std::string &jsonFileName,  // simparams.json
                  const bool verbose,
                  const bool warning);
```

**Key Methods:**
- `parseDoc()` - Parse kinetic data from simparams.json
- `parseKineticData()` - Route to appropriate kinetic model parser
- `makeModel()` - Create KineticModel instance for each phase
- `calculateKineticStep()` - Calculate dissolution for time step

### Lattice

**Purpose:** Holds the 3D microstructure as a cubic array of voxels.

**Constructor:**
```cpp
Lattice(ChemicalSystem *cs,
        RanGen *rg,
        int seedRNG,
        const std::string &initMicName,  // Microstructure file
        const bool verbose,
        const bool warning);
```

**Key Properties:**
- `xDim_`, `yDim_`, `zDim_` - Dimensions in voxels
- `resolution_` - Voxel size in micrometers
- `site_` - Vector of Site objects (one per voxel)
- `interface_` - Vector of Interface objects (one per phase)

### Controller

**Purpose:** Orchestrates the simulation loop.

**Constructor:**
```cpp
Controller(Lattice *msh,
           KineticController *kc,
           ChemicalSystem *cs,
           ThermalStrain *thmstr,
           const int simtype,
           const std::string &jsonFileName,  // simparams.json
           const std::string &jobname,
           const bool verbose,
           const bool warning,
           const bool xyz);
```

**Main Loop Method:**
```cpp
void doCycle(double elemTimeInterval);  // elemTimeInterval typically 24.0e-5 hours
```

**Cycle Steps:**
1. Kinetic dissolution calculation
2. GEMS thermodynamic equilibration
3. Microstructure update (growth/dissolution)
4. Output statistics

## Kinetic Models

### Model Types

| Type | Class | Phases | Use Case |
|------|-------|--------|----------|
| `ParrotKilloh` | ParrotKillohModel | Alite, Belite, Aluminate, Ferrite | Clinker phases |
| `Standard` | StandardKineticModel | Gypsum, Anhydrite, Hemihydrate | Sulfate phases |
| `Pozzolanic` | PozzolanicModel | Quartz, Mullite, fly ash glasses | Pozzolanic phases |

### ParrotKilloh Model Parameters

```json
{
  "type": "ParrotKilloh",
  "k1": 1.5,              // Nucleation/growth rate constant
  "k2": 0.05,             // Early diffusion rate constant
  "k3": 1.1,              // Late diffusion rate constant
  "n1": 0.7,              // Nucleation/growth exponent
  "n3": 3.3,              // Late diffusion exponent
  "dorHcoeff": 2.0,       // Lothenbach-Kulik H coefficient for critDOR
  "activationEnergy": 41570.0,  // J/mol
  "loi": 0.0              // Loss on ignition
}
```

**Default Parrot-Killoh Parameters:**

| Parameter | Alite | Belite | Aluminate | Ferrite |
|-----------|-------|--------|-----------|---------|
| k1 | 1.5 | 0.5 | 1.0 | 0.37 |
| k2 | 0.05 | 0.02 | 0.04 | 0.02 |
| k3 | 1.1 | 0.7 | 1.0 | 0.4 |
| n1 | 0.7 | 1.0 | 0.85 | 0.7 |
| n3 | 3.3 | 5.0 | 3.2 | 3.7 |
| Ea (J/mol) | 41570 | 20785 | 54040 | 34087 |

### Standard Model Parameters

```json
{
  "type": "Standard",
  "activationEnergy": 40000.0,      // J/mol
  "dissolutionRateConst": 5.0e-7,   // mol/m2/s
  "diffusionRateConstEarly": 5.0e-6,
  "diffusionRateConstLate": 5.0e-6,
  "dissolvedUnits": 2,              // DC units per dissolution
  "siexp": 1.0,                     // Saturation index exponent
  "dfexp": 1.1,                     // Driving force exponent
  "dorexp": 0.5,                    // Degree of reaction exponent
  "loi": 0.0
}
```

### Pozzolanic Model Parameters

```json
{
  "type": "Pozzolanic",

  "dissolutionRateConst": 1.4e-11,
  "diffusionRateConstEarly": 2.8e-12,
```

## simparams.json Schema

### Top-Level Structure

```json
{
  "environment": { ... },
  "microstructure": {
    "numentries": 38,
    "phases": [ ... ]
  },
  "time_parameters": { ... }
}
```

### Environment Section

```json
"environment": {
  "temperature": 298.15,          // Kelvin
  "reftemperature": 298.15,       // Reference temperature for kinetics
  "saturated": 1,                 // 1=saturated, 0=sealed curing
  "electrolyte_conditions": [
    {
      "DCname": "Ca(CO3)@",
      "condition": "initial",     // "initial", "fixed", or "attack"
      "concentration": 1.0e-6     // mol/kgw
    },
    ...
  ]
}
```

### Phase Entry Structure

```json
{
  "thamesname": "Alite",
  "id": 2,
  "cement_component": 1,              // 1=clinker, 0=other
  "gemphase_data": [ ... ],
  "kinetic_data": { ... },            // Optional
  "impurity_data": { ... },           // Optional
  "interface_data": { ... },          // Optional
  "display_data": { ... },            // Optional
  "poresize_distribution": [ ... ],   // Optional (for C-S-H)
  "Rd": [ ... ]                       // Optional (for C-S-H)
}
```

### gemphase_data

Links THAMES phase to GEMS phases and DCs:

```json
"gemphase_data": [
  {
    "gemphasename": "Alite",
    "gemdc": [
      { "gemdcname": "C3S" }
    ]
  }
]
```

For phases with multiple DCs (e.g., C-S-H):

```json
"gemphase_data": [
  {
    "gemphasename": "CSHQ",
    "gemdc": [
      { "gemdcname": "CSHQ-JenD", "gemdcporosity": 0.4935 },
      { "gemdcname": "CSHQ-JenH", "gemdcporosity": 0.4935 },
      { "gemdcname": "CSHQ-TobD", "gemdcporosity": 0.2004 },
      { "gemdcname": "CSHQ-TobH", "gemdcporosity": 0.2004 },
      { "gemdcname": "KSiOH", "gemdcporosity": 0.1825 },
      { "gemdcname": "NaSiOH", "gemdcporosity": 0.1825 }
    ]
  }
]
```

### impurity_data

Defines impurity coefficients for dissolution:

```json
"impurity_data": {
  "k2ocoeff": 0.00087,    // K2O coefficient
  "na2ocoeff": 0.0,       // Na2O coefficient
  "mgocoeff": 0.00861,    // MgO coefficient
  "so3coeff": 0.007942    // SO3 coefficient
}
```

### interface_data

Defines phase affinity for growth (contact angles):

```json
"interface_data": {
  "affinity": [
    { "affinityphase": "Alite", "contactanglevalue": 30 },
    { "affinityphase": "Belite", "contactanglevalue": 30 },
    { "affinityphase": "Portlandite", "contactanglevalue": 0 }
  ]
}
```

Contact angle values:
- **0** = High affinity (grows preferentially on this phase)
- **180** = Low affinity (avoids growing on this phase)

### display_data

RGB color for visualization:

```json
"display_data": {
  "red": 153.0,
  "green": 0.0,
  "blue": 0.0,
  "gray": 100.0
}
```

### poresize_distribution

For phases with internal porosity (e.g., C-S-H):

```json
"poresize_distribution": [
  { "diameter": 0.23228, "volumefraction": 0.01684 },
  { "diameter": 0.49089, "volumefraction": 0.02276 },
  ...
]
```

### Rd (Distribution Coefficients)

For alkali uptake in C-S-H:

```json
"Rd": [
  { "Rdelement": "K", "Rdvalue": 0.42 },
  { "Rdelement": "Na", "Rdvalue": 0.42 }
]
```

### time_parameters

```json
"time_parameters": {
  "finaltime": 56.0,    // Days
  "outtimes": [0.01, 0.1, 0.25, 0.5, 1, 3, 7, 14, 21, 28, 35, 42, 48, 54]
}
```

## Microstructure File Format

### Header Format (THAMES)

```
#THAMES:Version: 1.0
#THAMES:Image_Resolution: 1.0
#THAMES:X_Size: 200
#THAMES:Y_Size: 200
#THAMES:Z_Size: 200
```

### Voxel Data

After header, one integer per line representing phase ID at each voxel.
Ordering: z changes fastest, then y, then x.

```
Total voxels = X_Size * Y_Size * Z_Size
```

| File | Description |

## UI Integration Requirements

### Files to Generate

1. **simparams.json** - Main simulation parameters
   - Environment settings (temperature, saturation)
   - Phase definitions with GEMS linkages
   - Kinetic parameters per phase
   - Time parameters

2. **input.in** (optional) - Can pipe directly to stdin instead

### Data Sources in THAMES UI

| simparams.json Field | UI Source |
|---------------------|-----------|
| `environment.temperature` | Hydration setup UI |
| `environment.saturated` | Hydration setup UI |
| `environment.electrolyte_conditions` | Default + optional override |
| `microstructure.phases[].id` | PhaseIdMapping (from micgen) |
| `microstructure.phases[].thamesname` | PhaseIdMapping |
| `microstructure.phases[].gemphase_data` | GEMSParserService |
| `microstructure.phases[].kinetic_data` | Material database + UI |
| `microstructure.phases[].impurity_data` | ClinkerExtension |
| `microstructure.phases[].interface_data` | Defaults (configurable) |
| `microstructure.phases[].display_data` | PhaseColorService |
| `time_parameters.finaltime` | Hydration setup UI |
| `time_parameters.outtimes` | Hydration setup UI |

### Recommended Implementation

1. **HydrationInputService** - Generates input files
   - Method: `generate_simparams_json(mix_design, phase_mapping, options)`
   - Method: `generate_input_file(operation_path)`

2. **HydrationService** - Runs simulation
   - Method: `run_hydration(operation, callback)`
   - Handle stdin piping or input.in file
   - Monitor progress via output files

3. **Hydration Setup UI**
   - Temperature input (default: 298.15 K = 25°C)
   - Curing condition (saturated/sealed)
   - Final time (days)
   - Output times (list or interval)
   - Advanced: kinetic parameter override
