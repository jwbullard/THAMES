# Session 48 Summary: THAMES Architecture Triangle Slide Graphic

**Date:** June 7, 2026 — macOS

## Goal

Produce a single PowerPoint-suitable graphic illustrating the three pillars of THAMES (GEMS3K thermodynamics, kinetic constraints, 3D microstructure) and their bidirectional coupling. Pedagogically clear, artistically engaging, and reusable for slides AND papers without re-authoring.

## Final Output

Two theme variants of the same figure, both transparent-background SVGs at 1200×1000 viewBox:

- `docs/thames-architecture-triangle-dark.svg` — cream text and arrows for placement on dark slide backgrounds (e.g. Jeff's `#121217`).
- `docs/thames-architecture-triangle-light.svg` — dark-charcoal text and arrows for placement on white pages / light slides.

Identical panel content between themes; only the exterior text and arrow colors flip. Panel-interior text stays cream in both versions because the colored panel backgrounds are saturated enough to read against either page color.

## Composition

- **Layout.** Triangle: microstructure panel on top, GEMS bottom-left, kinetics bottom-right. Each edge of the triangle is a "lens" of two unidirectional Bezier curves — six curves total — with one arrowhead per curve. Bidirectional double-arrows were rejected during design discussion because they strip out the explicit direction-of-flow information that the lens design conveys.
- **Color assignment** (after a mid-session swap of microstructure ↔ kinetics):
  - 3D Microstructure: ember `#E07A2C`
  - GEMS3K Thermodynamics: deep teal `#397B8A`
  - Kinetic Constraints: jade `#5FAA82`
  - Background text: cream `#EDE4D3` (dark theme) or dark charcoal `#1A1A1A` (light theme)
- **Source-color flow labels.** Each flow label is rendered in the color of the panel the data leaves. A viewer can trace any label back to its source without reading the arrowhead direction. Mapping:
  - "phase volumes" + "saturation index Ω" → teal (sourced from GEMS)
  - "rate → voxel update" + "DC upper limits" → jade (sourced from kinetics)
  - "surface areas" + "interface area" → ember (sourced from microstructure)
- **Panel content:**
  - Microstructure: real ThamesRender PNG of `HY-ccr152-ws45` at 12 h (`frames/frame_0004_0d_12h00m.png`), downsampled from 1920×1440 to 520×390 (retina-safe for the 260×180 display) and base64-inlined.
  - GEMS: schematic 3D Gibbs energy surface with a clearly-marked minimum (`G_min` in ember). Rendered by `docs/scripts/make_gibbs_surface.py` (matplotlib magma colormap, transparent background).
  - Kinetic: shrinking-particle schematic (cream particle "C₃S" with inward dissolution arrows on a dashed original-surface circle) above the standard THAMES rate equation `R = k A (1 − Ω^p)^q`. The equation is a matplotlib-mathtext PNG, not SVG text — Affinity could not be coerced into rendering the superscripts correctly via any combination of tspan `dy`, `baseline-shift`, or absolute-y attributes.
- **Drop-shadow elevation.** A subtle `feDropShadow` filter on each panel rect lifts the cards off the background. Opacity 0.45 on dark theme, 0.22 on light, so the shadow is visible against either page color.

## Builder script

All artifacts are generated by `docs/scripts/build_triangle_svgs.py`. The script is the source of truth — both SVGs are stale outputs to be re-generated rather than edited directly. The script:

1. Loads the microstructure PNG, downsamples it to ≤520 px, and base64-encodes.
2. Loads the Gibbs surface PNG and base64-encodes.
3. Renders the rate equation via matplotlib mathtext (`render_equation_png` inline function) and base64-encodes.
4. Emits the SVG template, parameterised by theme (`dark` or `light`), with curve geometry / label positions / source-color hex values all defined as module-level constants.

Running `~/Code/Python/Envs/Default/bin/python docs/scripts/build_triangle_svgs.py` regenerates both SVGs from current source data.

The Gibbs surface PNG itself is regenerated by `docs/scripts/make_gibbs_surface.py` — kept separate because the surface composition formula is non-trivial and worth editing independently.

## Affinity Designer SVG compatibility — three battles

Affinity has selective SVG-CSS support. Every cycle of iteration exposed another behavior, and the fixes are baked into the final builder so future regenerations don't reintroduce the bugs.

1. **`tspan` superscripts (`dy`, `baseline-shift`, absolute-y) all mishandled.** Affinity rendered `R = k A (1 − Ω^p)^q` with the `p` and `q` glyphs at the parent baseline regardless of which positioning strategy was tried. Final fix: render the equation as a flat PNG via matplotlib mathtext and inline it as `<image>`. Bulletproof, at the cost of the equation no longer being live-editable as text inside Affinity.
2. **CSS `font-family` not inherited from root `<svg>`.** Inline `font-family="Inter"` on the root SVG element should cascade per spec, but Affinity does not implement that inheritance. CSS rules in a `<style>` block are also ignored for `font-family` specifically. Final fix: emit `font-family="Inter"` as an inline attribute on every individual `<text>` element. The builder applies this via the SVG template.
3. **CSS `fill` rule for `.label-teal` silently dropped.** A class named after a CSS color keyword (`teal`) collided with Affinity's CSS parser. Renamed all source-color classes to source-based names (`.src-gems`, `.src-micro`, `.src-kinetics`) and ultimately moved the colors to inline `fill=` attributes on the labelled tspans, removing CSS color dependence entirely.

GIMP and modern browsers rendered the SVG correctly throughout — the issues were Affinity-specific.

## Iteration log

Roughly the order changes landed:

1. Drafted the geometry with two themes (transparent backgrounds, jade/teal/ember palette).
2. Generated the Gibbs energy surface PNG via matplotlib (`make_gibbs_surface.py`).
3. Inlined external PNGs as base64 because Affinity refused to load `href` references.
4. Fixed rotated-text positioning (curve midpoints + perpendicular offset; switched to single-`<text>`-with-`<tspan>` idiom; bumped fonts to 18/15 px).
5. Replaced the central "THAMES" hub with empty space (it was decorative, not pedagogical).
6. Updated the rate equation form to the literal THAMES Standard model: `R = k A (1 − Ω^p)^q`.
7. Swapped microstructure ↔ kinetics panel colors per user preference.
8. Switched equation from SVG tspan superscripts to matplotlib-mathtext PNG (Affinity workaround).
9. Swapped to a better microstructure frame (`HY-ccr152-ws45` at 12 h instead of the t=0 packing).
10. Added source-color binding to all six flow labels.
11. Added `feDropShadow` to all three panels (theme-dependent opacity).
12. Renamed CSS classes off the `teal` collision; ultimately moved colors fully inline.
13. Added `font-family="Inter"` inline to every `<text>` element (Affinity inheritance workaround).
14. Reduced panel-title font from 24 → 22 px so "GEMS3K Thermodynamics" fits the panel width with Inter.
