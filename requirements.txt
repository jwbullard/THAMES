# --------------------------------------------------------------------------
# Windows (MSYS2) note
# --------------------------------------------------------------------------
# On Windows we run inside thames-env-windows, a venv created with
# --system-site-packages against MSYS2's mingw-w64 Python. Most of the
# packages below (PyGObject, SQLAlchemy, pandas, numpy, pydantic, PyYAML,
# Pillow, matplotlib, pyvista 0.36, reportlab, openpyxl, lxml, ...) are
# already provided by pacman and are visible via system-site-packages.
#
# Running `pip install -r requirements.txt` on Windows will often try to
# UPGRADE those MSYS2-supplied packages (e.g. pyvista 0.36 -> 0.47) and
# cascade into source builds (numpy / meson-python / ninja) that fail
# because MSYS2 does not ship ninja by default. For the alpha release,
# install only the deps that MSYS2 does NOT provide:
#
#     pip install "markdown>=3.4.0"
#
# That is the only pip-managed dependency introduced by Session 41. All
# other imports the app actually uses are already satisfied by MSYS2.
# See docs/ALPHA_RELEASE_PREPARATION.md Step 2.
# --------------------------------------------------------------------------
